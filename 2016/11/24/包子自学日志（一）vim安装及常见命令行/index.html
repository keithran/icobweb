包子自学日志（一）vim安装及常见命令行

BaoZi's Learning Journal(1) Install vim & The Command line of Ubuntu

Posted by Keith Ran on 2016-11-24

以前根本没接触过Linux操作系统，类生系统就玩过IOS，所以今天总结的Ubuntu的命令行，也是我从零学起的小结。

因为工作的关系，我工作的一年半中基本没敲打过什么代码，每天都是产品的工作，需求调研、准备路演材料、设计原型、写PRD文档等等。其实在这些过程中，我也学到不少东西，但总觉得少了一点什么。直到有一天我在PMCAFF上看到一段文字：“懂不懂技术并不是评价一个产品经理优劣的标准，但是多一项技能未尝不是一件好事”。打动我的不是这段文字有多么的感染力。

而是它提醒了我：不要止步，继续向前。

开始我的自学日志：vim安装及常见命令行

其实我一开始自学的是Python，想分享的也是学习Python过程中的学习笔记，但是被一个大神引领到了奇怪的领域，所以开始并发式的学习，反正都有互相促进的地方。

首先安装Ubuntu系统

版本：Ubuntu14.04
我知道现在大家基本都是WIN／IOS系统，所以先装Ubuntu。但是，我选择直接跳过这步骤···想分享的小伙伴直接百度😶

安装vim

版本：7.4.52
我选择使用vim作为我的主要文本编辑器（大神推荐，不过用上手了，的确挺棒的）。这款文本编辑器有3大特点：

  • 不设置一下，真的不能用······
  • 设置完成后，瞬间变身一款超轻巧强大的IDE
  • 可玩性较高，学vim，我学了好久······

首先使用Ubuntu的终端进行vim的安装。通过安装vim，顺便分享一下常见的命令行输入。


sudo：最高权限，一般通过终端安装软件，都是需要打上这行命令进行申明最高权限。
apt-get：Advanced Package Tool 进行deb包的管理操作。
install vim：这个不用说了吧，install 是 安装的方法，vim是目标软件。

常见的操作

update：更新软件包版本；
install：安装；
remove：卸载+保存配置文档；
remove –purge：卸载+删除配置文件；
autoremove packagename：删除和相关依赖包；
autoremove –purge packagename：彻底删除；

关联知识点1：dpkg –force-all –purge packagename：强制删除（如在win上删除金山毒霸，强制删除，可能会有风险、慎用）。

autoclean：apt会把已装或已卸的软件都备份在硬盘上，该命令删除软件备份；
clean：删除所有的安装包；
upgrade：更新所有的安装包；
dist-upgrade：更新系统至最新版本；

关联知识点2：超喜欢这个功能，Tab自动补全命令，例如我输入apt-get，点击TAB，会出现相关操作列表，命令不用记太全。

apt-file search filename：查询一个特定的软件包；

其实在安装vim之前，还是先多多熟悉Ubuntu命令行，日后这可以是不离身的重型武器。今天的知识点就先梳理到这里，自学日志（二）来进行vim的配置和一些基础语法的分享。
