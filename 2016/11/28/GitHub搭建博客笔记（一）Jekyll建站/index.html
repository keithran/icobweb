GitHub搭建博客笔记（一）Jekyll建站

Building your blog on the GitHub - by Jekyll

Posted by Keith Ran on 2016-11-28

在GitHub下搭建属于自己的博客

一般在GitHub下建博一般都采用以下两种方式，其他的方式也有，但是我没怎么关注到，有的朋友可以留言分享一下。

  • 利用建站的形式进行博客建立（我采取的是这一种）P_Chou Tech Space
  • 利用静态界面进行建立（比较简单&便捷）isnowfy

在搭建自己的博客时，不想单纯的使用静态的界面的形式进行管理，并且想保证实时更新与一些文件处理上的管理，所以开始使用Git来进行管理，减少人工。

GitHub需要掌握：node与git的使用

git的管理原理，讲实话我也不太懂，也是按照搜索到的信息一点点的学。按照现在的理解来说，就是在本地做了一个版本的发布器与底层仓库，然后连接云端进行数据交互的一种方法。

#暂时这么理解，有大牛一定要帮我佐证！

安装node与git,使用终端进行安装即可

2. 初始化Git仓库

3. 生产秘钥

这个东西就是连接本地与github的媒介，需要通过本地生产一个唯一的秘钥，然后在github上绑定。

如果检测出没有秘钥的话，那说明是第一次使用git，就需要生产新的ssh秘钥。

4.绑定github

Key的位置输入完整的SSH Key，可通过终端打开之前生产的id_rsa文件进行查看。

现在就可以实现本地通过SSH进行远程登录与远程执行命令。

5.设置域名

通过以上的步骤，你已经实现了在github上搭建博客的基础步骤了。
但是！人生没个追求，还活个屁呀。我就想通过自己独特的域名访问自己的博客！

那实现的过程也很简单，买个个性的域名先。
我在godaddy上买了一个自己喜欢的域名icobweb.com。

i cobweb:
我的蜘蛛网，像蜘蛛一下织网捕蝶，在风雨中捕捉到自己的眼中的独特，我会将我学习笔记和自己的设计Demo都分享到这个平台，希望能找到一群志同道合的knowledge sharers。

管理器：DNSPOD
首先在GoDaddy中，将域名转入到DNSPOD的伺服器下。
（其实GoDaddy的后台管理也可以做DNS管理，但是访问速度实在令我抓狂，还是代理管理吧）

再通过DNSpod将DNS指向到github的服务器上

现在就可以通过icobweb.com这个域名访问到我的github博客，也可以通过keithran.github.io这个域名进行访问。

搭建一个简单的博客

基础的工作都做好了，现在我们可以开始搭建一个简单的博客项目了。

1.在本地搭建一个博客的仓库

新建一个目录（项目名称随意）

这个目录作为项目的主目录，对目录进行git初始化操作

创建一个没有父节点的分支 gh-pages。因为github已经规定，只有在这个分支下的页面才会生产网页文件，之后的所以动作都是在这个分支下完成的。

2.创建设置文件


_configh.yml：jekyll的设置文件。
在文件中填写如下内容：
baseurl: /jekyll_demo

index.html：首页

使用了{% for post in site.posts %}，表示对所有帖子进行一个遍历
_layouts/default.html：用于存放模板文件，暂时采用一下的默认模板：

_post/2016-11-1-first-test.md：创建文章，可以采用html/md两种格式的文档

具体的模板架构，大家可以参阅Jekyll的官方社区
